# HexaForge - Game Design Requirements Document

## Overview

HexaForge is a factory idle/incremental game originally based on Factory Idle. This document defines the changes needed to make HexaForge a distinct game with its own identity, visual style, and unique gameplay mechanics while retaining the core factory-building idle loop.

---

## Phase 1: Identity & Visual Overhaul (Current Sprint)

### D1 - Remove Original Branding
Remove all references to "Factory Idle", factoryidle.com, and associated branding.
- **Files**: `intro.html`, `help.html`, `purchases.html`, `config.js`, `Play.js`, `httpMigrate.js`, `README.md`, `SETTINGS_IMPLEMENTATION.md`
- **Scope**: Replace game title, remove external URLs, remove original support email, update all user-facing strings

### D2 - Rename Storage Keys
Rename all localStorage/cloud keys from `FactoryIdle*` to `HexaForge*`.
- `FactoryIdleUserHash` -> `HexaForgeUserHash`
- `FactoryIdleLocal` -> `HexaForgeLocal`
- `FactoryIdleSave` -> `HexaForgeSave`

### B2 - Modern UI Overhaul
Replace the dated table-based, flat-black UI with a modern dark theme using:
- CSS custom properties (variables) for consistent theming
- Modern font stack (Inter / system fonts replacing Verdana)
- Glassmorphism panels (semi-transparent backgrounds + backdrop blur)
- Consistent border-radius, spacing, and shadow system
- Smooth transitions on all interactive elements
- Preserve all existing functionality and layout structure

### B3 - New Color Palette
Replace the industrial yellow/cyan palette with a modern scheme:
- **Background**: Deep navy (`#0a0e17`) instead of pure black
- **Panels**: Translucent dark blue-gray with blur
- **Money/Credits**: Warm amber (`#fbbf24`)
- **Research**: Sky blue (`#38bdf8`)
- **Success/CTA**: Emerald (`#34d399`)
- **Danger**: Coral (`#f87171`)
- **Borders**: Subtle blue-tinted glows

## Phase 2: Economy Rework

### Four-Currency System
Replace the two-currency system (money + research) with four currencies:

| Currency | Role | Source |
|----------|------|--------|
| **Credits** | Primary currency (replaces money) | Selling goods via any seller type |
| **Research** | Unlocks tech tree | Research Centers + Labs (unchanged) |
| **Unrest** | Risk meter (0-10000) | Generated by Military sellers, reduced by Local sellers |
| **Influence** | Strategic currency | Generated by Export sellers, spent on one-time purchases |

### Three Seller Types
Replace the single `Seller` strategy with three variants:

**Military Seller**
- Highest credit yield per resource
- Generates Unrest per sale cycle
- Best for rapid income when Unrest is manageable

**Local Seller**
- Lowest credit yield per resource
- Reduces Unrest per sale cycle
- Stabilizer for balancing Military output

**Export Seller**
- Moderate credit yield
- Generates Influence instead of Unrest
- Influence used for: land purchases, one-time upgrades, unrest reduction

### Unrest System
- Unrest accumulates from Military sellers, decays from Local sellers
- **Soft cap model** (not hard loss):
  - 0-3000: No effect
  - 3000-6000: Production efficiency penalty (scales linearly)
  - 6000-8000: Severe penalties + random machine shutdowns
  - 8000+: Military sellers auto-pause, warning displayed
  - Natural decay over time prevents idle-period losses
- Influence can be spent to instantly reduce Unrest

### Influence Spending
- Purchase new factory land (alternative to credits)
- One-time permanent upgrades (unique to Influence)
- Emergency Unrest reduction
- Unlock special components (Power Pylon, etc.)

## Phase 3: New Mechanics

### Power Pylon Component
- New component type: `powerPylon`
- Provides an Area of Effect (AoE) buff to surrounding tiles
- Effect: Reduces `DelayedAction` interval for machines in range (faster production)
- Effect on transport: Reduces `TransportSpeedModel` slowdown within range
- Upgradeable: Higher levels = larger AoE + stronger buff
- Requires research to unlock

### Resource Refinement
- Certain resources can pass through converters multiple times
- Each refinement tier increases sell value (e.g., Iron -> Refined Iron -> High-Grade Iron)
- Requires specific converter types per tier
- Creates longer, more complex logistics chains

### Multi-Stage Converters
- New converter components that perform multiple sequential operations
- Example: Tank Assembler takes hull + turret + engine in staged process
- Internal buffer between stages visible to player
- Higher throughput than chaining individual converters but more expensive

### Dedicated Lab Map
- New factory map exclusively for research
- Unique terrain and component set
- Only place to access high-tier research
- Research Centers here get efficiency bonuses
- Unlocked via Influence or late-game progression

## Technical Notes

### Existing Systems to Leverage
- `TransportSpeedModel.js` already has network slowdown - enhance tuning knobs
- `Seller.js` has `sellPrice`, `sellMargin`, `bonus`, `requiresResearch` - extend with `sellerType`
- `DelayedAction.js` manages intervals - Power Pylon modifies these
- `FactoryCalculator.js` tick phases (input -> transport -> output) accommodate new currency flows
- Multiple factory maps already supported via config - Lab map is config-driven

### Save Compatibility
- New currencies and components require save format version bump
- Migration path from old saves should zero-initialize new fields
